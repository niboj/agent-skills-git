# Skill Authoring Guidelines

## Authoring goals

- keep each skill narrowly scoped ;
- express activation intent in the `description` field ;
- make the ordered procedure directly executable by an agent ;
- surface guardrails before risky steps ;
- move explanation depth to `references/`.

## Required structure

- `SKILL.md`
- `README.md`
- `references/concepts.md`
- `references/decision-rules.md`
- `references/examples.md`
- `assets/examples/sample-scenarios.md`
- `assets/templates/` only when reusable artifacts exist
- `scripts/` only when the automation adds repeatable value
