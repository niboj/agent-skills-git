# Naming Conventions

## Skill names

- lower-case letters, digits and hyphens only ;
- start with the domain, here `git-` ;
- prefer action-oriented names.

## File names

- `concepts.md` for shared concepts within a skill ;
- `decision-rules.md` for explicit branching or safety criteria ;
- `examples.md` for numbered example records ;
- `sample-scenarios.md` for longer end-to-end walkthroughs.
